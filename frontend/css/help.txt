CSS DIRECTORY - STYLES AND THEMING

PURPOSE:
This directory contains all stylesheets for the website.

FILE: style.css
- The main and ONLY stylesheet (single source of truth)
- Contains the dark mode color palette
- Includes responsive design breakpoints
- Has all component styles (navbar, buttons, cards, footer, etc.)
- Uses CSS Flexbox and Grid for layouts
- No external CSS frameworks (pure vanilla CSS)

STRUCTURE OVERVIEW:
- Color variables for dark mode theme
- Reset/normalize styles
- Typography and base styles
- Component styles (navbar, hero, products, footer, etc.)
- Responsive media queries
- Utility classes

DEVELOPER TASKS:
1. Never create multiple CSS files - keep everything in style.css
2. Use CSS custom properties (--color-*, --spacing-*, etc.)
3. Maintain consistent spacing using the spacing scale
4. Add new styles for new pages following the existing pattern
5. Test dark mode appearance across all browsers
6. Verify responsive design on mobile, tablet, and desktop
7. Use Flexbox and CSS Grid for layouts (no floats)

BEST PRACTICES:
- Update color variables when changing the theme
- Keep selectors specific but not too deeply nested
- Comment section headers clearly
- Group related styles together
- Test all hover/active/focus states
