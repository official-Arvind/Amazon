JAVASCRIPT DIRECTORY - APPLICATION LOGIC

PURPOSE:
This directory contains all vanilla JavaScript files for the website.

FILES:
- main.js: Main entry point with all app logic

STRUCTURE:
main.js includes:
- DOM event listeners and initialization
- Navigation interactions
- Product management and filtering
- Shopping cart functionality (will use Firebase later)
- User authentication (will use Firebase later)
- Utility functions and helpers

DEVELOPER TASKS:
1. Keep all logic in main.js for now (monolithic approach)
2. Use vanilla JavaScript only (no jQuery, no frameworks)
3. Add comments and organize code into sections
4. Test all interactions thoroughly
5. Ensure mobile touch events work properly
6. Add error handling for API calls (Firebase)
7. Use localStorage for cart persistence (before Firebase)

IMPORTANT:
- No external JavaScript libraries or frameworks
- Write semantic, readable code
- Use modern ES6+ syntax (const, let, arrow functions, etc.)
- Always test in the browser console for errors
- Keep functions small and reusable
- Document complex logic with comments

FUTURE INTEGRATION:
- Firebase authentication will go in a separate firebase-config.js
- Backend API calls will be handled here
- Service workers for offline functionality can be added later
